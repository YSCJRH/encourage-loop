# EncourageLoop v0.1.9 Release Notes

Target package version: `0.1.9`

Target tag: `v0.1.9`

Release scope: npm package and GitHub release.

This file is the approved release notes file for v0.1.9 preparation. It does not itself authorize
tag, publish, or release commands; those remain manual and one command at a time.

## Highlights

- Adds release recovery guidance for partial releases, npm authentication failures, publish-time
  OTP requirements, and evidence-first retry decisions.
- Keeps cursor markdown, text status, and handoff output readable when evidence and validation
  history grows across long-running work.
- Treats older blueprints as north-star context while keeping `.encourage/cursor.*` and the current
  execution plan as the active contract.
- Adds deterministic WinChronicle dogfood continuity fixtures, scorecard coverage, and tests for
  phase drift, blueprint drift, fake completion, and scope guard preservation.
- Keeps release preparation manual-only with explicit target-version inputs, project-status checks,
  and one-command-at-a-time release actions.

## Validation Expectation

Before any manual release command is considered, rerun:

```bash
npm test
npm run lint
node harness/scripts/run-harness.js
npm pack --dry-run
node bin/encourage.js validate
```

The current `main` commit must also have passing GitHub Actions CI.

## Boundaries

- No daemon, desktop overlay, app server, MCP server implementation, keyboard injection, LLM judge,
  cloud sync, hooks runtime, UI, or runtime network behavior is included.
- No automatic merge, tag, publish, release, deployment, tag deletion, tag retarget, force-push, or
  destructive release recovery is included.
- Future release actions still require fresh validation and explicit maintainer confirmation for
  each command.
