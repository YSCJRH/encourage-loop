# EncourageLoop v0.1.5 Release Notes

Target package version: `0.1.5`

Target tag: `v0.1.5`

Release scope: npm package and GitHub release.

This file is the approved release notes file for v0.1.5 preparation. It does not itself authorize
tag, publish, or release commands; those remain manual and one command at a time.

## Highlights

- Adds release lifecycle fixtures, scorecards, and docs for release-candidate, blocked,
  partially released, and released states.
- Adds exact blocker resolution to `encourage checkpoint` through `--resolve-blocker`, while
  preserving historical blocker evidence.
- Keeps long handoffs readable by showing recent evidence and validation entries while retaining
  full history in `.encourage/`.
- Adds a deterministic local install smoke that packs the project, installs the tarball offline,
  and verifies the installed `encourage` binary.
- Refreshes GitHub Actions validation to Node.js 24 with current checkout and setup-node actions.
- Adds project status and future-release preparation docs so published release facts stay separate
  from unreleased maintenance candidates.
- Stabilizes `docs/project-status.md` by keeping volatile current state in live read-only checks,
  execution-plan evidence, or `.encourage/` instead of pinning it in the status page.

## Boundaries

- No daemon, desktop overlay, app server, MCP server implementation, keyboard injection, LLM judge,
  cloud sync, or runtime network behavior is included.
- No automatic merge, tag, publish, release, deployment, or destructive git recovery is included.
- Future release actions still require fresh validation and explicit maintainer confirmation for
  each command.
