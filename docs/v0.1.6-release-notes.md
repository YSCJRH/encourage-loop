# EncourageLoop v0.1.6 Release Notes

Target package version: `0.1.6`

Target tag: `v0.1.6`

Release scope: npm package and GitHub release.

This file is the approved release notes file for v0.1.6 preparation. It does not itself authorize
tag, publish, or release commands; those remain manual and one command at a time.

## Highlights

- Keeps `node bin/encourage.js status` readable when validation history is long by rendering the
  latest validation entries as a bounded list instead of one long line.
- Shows how many validation entries are hidden from text status output and states that older
  entries remain in `.encourage/` history.
- Preserves the full `last_validation` array for `getStatus()` and `encourage status --json`
  consumers.
- Adds focused status tests for long validation history and JSON status preservation.
- Records v0.1.6 status-readability readiness through local validation and GitHub Actions evidence.

## Boundaries

- No daemon, desktop overlay, app server, MCP server implementation, keyboard injection, LLM judge,
  cloud sync, or runtime network behavior is included.
- No automatic merge, tag, publish, release, deployment, or destructive git recovery is included.
- Future release actions still require fresh validation and explicit maintainer confirmation for
  each command.
