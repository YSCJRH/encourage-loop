# EncourageLoop v0.1 Readiness Review

## Status

EncourageLoop v0.1 has been released.

The MVP remains a deterministic Node.js CLI, repo-scoped Codex skill, plugin manifest,
local marketplace example, fixtures, tests, scorecards, and future-integration docs.

The release path remained manual-only.
Maintainer release authorization was given.
The `v0.1.0` tag, npm publication, and GitHub release were completed manually.

Remote CI has passed for the released commit.

## Changed files

- `package-lock.json`: enables `npm ci` locally and in GitHub Actions.
- `.github/workflows/ci.yml`: runs `npm ci`, tests, lint, harness, and package dry-run.
- `src/cli.js` and `src/checkpoint.js`: preserve repeated `--validation` checkpoint flags.
- `src/rules.js` and `src/validate.js`: check dangerous commands only in structured command fields.
- `schemas/event.schema.json`: adds optional `command` for future local state events.
- `AGENTS.md`, `START_CODEX.md`, `prompts/codex-startup.md`, and both `SKILL.md` copies:
  remain readable multi-line Markdown.
- `skills/encourage-loop/SKILL.md` and `.agents/skills/encourage-loop/SKILL.md`:
  stay byte-for-byte synchronized with valid YAML frontmatter.
- `tests/cli-smoke.test.js`: covers repeated checkpoint validation flags through the CLI.
- `tests/validate.test.js`: covers command-field RED detection without prose false positives.
- `tests/packaging.test.js`: covers plugin JSON, skill frontmatter, skill sync, package allowlist,
  CLI bin manifest normalization, and CI workflow commands.
- `tests/docs-contract.test.js`: covers readable multi-line operator Markdown.
- `docs/release-checklist.md`: documents the maintainer-only release gate and manual commands.
- `package.json`: uses the npm-normalized CLI bin path `bin/encourage.js`.
