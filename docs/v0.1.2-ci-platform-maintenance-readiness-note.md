# EncourageLoop v0.1.2 CI Platform Maintenance Readiness Note

## Status

EncourageLoop has prepared a v0.1.2 CI platform maintenance candidate.

This note is readiness evidence only. It does not publish a package, create a tag, create a
GitHub release, merge a branch, deploy anything, or authorize automation to do those actions.

## Completed maintenance

- CI now uses `actions/checkout@v6` and `actions/setup-node@v6`.
- CI now runs local validation commands on Node.js 24.
- CI still runs `npm ci`, `npm test`, `npm run lint`, `node harness/scripts/run-harness.js`,
  and `npm pack --dry-run` in order.
- Packaging tests now pin the intended CI action versions, Node.js runtime, and manual-only release
  boundary.
- The workflow does not use `FORCE_JAVASCRIPT_ACTIONS_TO_NODE24`, tag commands, publish commands,
  GitHub release commands, deploy commands, or force flags.

## Platform context

- GitHub announced that GitHub Actions runners begin moving JavaScript actions to Node.js 24 by
  default on June 2, 2026:
  <https://github.blog/changelog/2025-09-19-deprecation-of-node-20-on-github-actions-runners/>.
- `actions/checkout@v6` documents Node.js 24 runtime support:
  <https://github.com/actions/checkout>.
- `actions/setup-node@v6` documents Node.js 24 usage with npm caching:
  <https://github.com/actions/setup-node>.
