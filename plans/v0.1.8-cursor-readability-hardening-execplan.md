# EncourageLoop v0.1.8 Cursor Readability Hardening ExecPlan

## Summary

After the v0.1.6 and v0.1.7 maintenance rounds, `node bin/encourage.js status` and release
recovery docs are bounded and readable, but `.encourage/cursor.md` itself can still become very
large during long-running work because it renders the full evidence and validation arrays.

The goal is to make the human-readable cursor markdown stay concise without deleting evidence or
weakening machine-readable state. Keep the full structured history in `.encourage/cursor.json` and
state JSONL files. Do not rewrite historical `.encourage/` logs, add a daemon, add network runtime
behavior, add release automation, or perform any release action.
