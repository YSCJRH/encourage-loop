# EncourageLoop v0.1.5 Status Stability ExecPlan

## Summary

Keep `docs/project-status.md` useful without making it self-expire after every evidence commit.

The project status page should record stable release and maintenance-candidate facts. Fast-moving
facts such as current `HEAD`, current cursor, and GitHub Actions run IDs belong in `.encourage/`,
execution-plan evidence, or direct read-only checks.

The original goal is documentation stability, not a package release. Do not tag, publish, create a
GitHub release, merge automatically, bump a version, or add runtime features unless a later
release-preparation addendum explicitly narrows and authorizes tracked-file preparation changes.
