# EncourageLoop v0.1 MVP ExecPlan

## Summary

Build the smallest useful EncourageLoop package: a deterministic Node.js CLI, a Codex skill,
a plugin manifest, repo-scoped skill loading, local marketplace metadata, and blueprints/prompts
that let Codex App continue work without re-planning the entire project each turn.

This plan is the execution contract. Older research and blueprints are boundary context, not the per-turn script.

## Execution Cursor

- Current baseline: new open-source repository at `https://github.com/YSCJRH/encourage-loop`
- Target: v0.1.0 MVP package
- Current stage: R4
- Stage status: release_candidate
- Last completed evidence: release-candidate hardening added `package-lock.json`, CI, stronger
  packaging/docs contracts, command-field-only dangerous command detection, repeated checkpoint
  validation support, and readable synchronized skill docs.
- Last validation: `npm ci`, `npm test`, `npm run lint`, `node harness/scripts/run-harness.js`,
  `npm pack --dry-run`, and `node bin/encourage.js validate` pass locally.
- Next atomic task: maintainer reviews CI/package evidence and explicitly decides manual tag/publish/release.
- Known blockers: none
- Scope guard:
  - no daemon
  - no keyboard injection
  - no LLM calls
  - no automatic merge/release
  - no abuse/whip branding
